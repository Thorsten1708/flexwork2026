🏌️ Projektübersicht

Die Meeting@FlexWork Golf App ist eine Progressive Web App (PWA) zur Verwaltung und Auswertung des jährlichen Meeting@FlexWork Golf-Turniers. Sie läuft vollständig im Browser, ist auf dem Smartphone installierbar und synchronisiert Ergebnisse in Echtzeit zwischen allen Geräten.

Live-URL: https://thorsten1708.github.io/flexwork2026/

🔥 Firestore-Struktur

Die App verwendet zwei Pfad-Schemata – einen Legacy-Pfad für Saison 2026 und einen neuen hierarchischen Pfad für alle anderen Saisons:

# Saison 2026 (Legacy-Pfad – wird nicht geändert)
tournaments/
  mfw2026/
    qualiRounds: Array
    finalRounds: Array

# Alle anderen Saisons
seasons/
  {year}/
    tournaments/
      mfw{year}/
        qualiRounds: Array
        finalRounds: Array

Datenstruktur einer Runde

{
  "name": "Bad Griesbach - Uttlau",   // Platzname
  "date": "14.09.25",                  // Datum
  "fixed": true,                       // false = editierbar in der App
  "netto": [22, 23, 26, 23],          // [Lutz, Markus, Henrick, Thorsten]
  "brutto": [13, 8, 11, 9]            // [Lutz, Markus, Henrick, Thorsten]
}
Hinweis: Wert 0 bedeutet "nicht gespielt". Die App berechnet dann automatisch einen Ersatz-Wert (schlechtester Wert der anderen Spieler dieser Runde).

🔐 Firestore Security Rules

Die Rules erlauben öffentliches Lesen und Schreiben – der Schutz erfolgt über den PIN in der App:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tournaments/{doc} {
      allow read, write: if true;
    }
    match /seasons/{season}/tournaments/{doc} {
      allow read, write: if true;
    }
  }
}

Rules anpassen: Firebase Console → Firestore → Rules

📅 Neue Saison anlegen

1
Saison in index.html registrieren

In der SEASONS-Konstante einen neuen Eintrag hinzufügen:

const SEASONS = {
  '2026': { label: 'Saison 2026', legacy: true, legacyDoc: 'mfw2026' },
  '2027': { label: 'Saison 2027', legacy: false, tournamentId: 'mfw2027' }, // NEU
  // ...
};
2
DEFAULT_QUALI_ROUNDS und DEFAULT_FINAL_ROUNDS anpassen

Plätze, Daten und bekannte Ergebnisse eintragen. Unbekannte Runden mit netto: [0,0,0,0] vorbelegen.

3
Versionsnummer erhöhen

const APP_VERSION = '4.0';

4
Pushen

Die App deployed automatisch via GitHub Pages.

git add index.html
git commit -m "v4.0: Saison 2027 hinzugefügt"
git push

🚢 Deployment

Die App wird automatisch über GitHub Pages deployed. Jeder Push auf den main-Branch löst ein Deployment aus (Dauer: ca. 1–2 Minuten).

cd meeting-flexwork-2026
git add .
git commit -m "v3.x: Beschreibung der Änderungen"
git push

Nach dem Push: Hard Refresh im Browser (Cmd+Shift+R) damit der Service Worker die neue Version lädt.

📋 Spielregeln

Qualifikationsrunde

  • Jeder Spieler spielt alle Runden der Saison
  • Streicher-Regel: Die schlechteste Netto-Runde jedes Spielers wird gestrichen und zählt weder für Netto noch für Brutto
  • Ersatz-Wert: Fehlt ein Spieler in einer Runde (Netto = 0), bekommt er automatisch den schlechtesten Netto-Wert der anderen Spieler dieser Runde als Ersatz
  • Bei mehreren Ersatz-Runden: Der schlechteste Ersatz-Wert wird als Streicher verwendet

Endrunde Race4Munich

  • 4 Runden in Bad Griesbach (Do–Sa)
  • Kein Streicher – alle 4 Runden zählen
  • Faktor ×2 in der Gesamtwertung

Gesamtwertung

Gesamt Netto  = Quali Netto gewertet + (Endrunde Netto × 2)
Gesamt Brutto = Quali Brutto gewertet + (Endrunde Brutto × 2)
